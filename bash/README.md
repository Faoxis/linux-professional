# Домашнее задание: Bash-скрипт для анализа access-лога

## Задание

Написать скрипт для CRON, который раз в час формирует отчёт по access-логу nginx и отправляет его на заданную почту.

### Требования

- **Top IP-адресов** — адреса с наибольшим числом запросов (с момента последнего запуска).
- **Top URL** — запрашиваемые URL с наибольшим числом запросов (с момента последнего запуска).
- **Ошибки** — ошибки веб-сервера и приложения: коды 4xx/5xx с указанием URL (с момента последнего запуска).
- **HTTP-коды** — все коды ответов с указанием количества (с момента последнего запуска).
- **Защита от параллельного запуска** — скрипт не должен допускать одновременную работу нескольких копий.
- **Временной диапазон** — в письме указан обрабатываемый период.

## Реализация

### Файлы

| Файл | Описание |
|---|---|
| `log_report.sh` | Основной скрипт анализа и отправки отчёта |
| `access-4560-644067.log` | Пример access-лога nginx для тестирования |

### Использование

```bash
# Запуск вручную (лог по умолчанию — access-4560-644067.log, почта — root@localhost)
./log_report.sh

# Запуск с указанием лог-файла и email
./log_report.sh /var/log/nginx/access.log admin@example.com
```

### Настройка CRON

```
0 * * * * /path/to/log_report.sh /var/log/nginx/access.log admin@example.com
```

### Как работает

1. **Блокировка** — используется `flock` (если доступен) или PID-файл для предотвращения параллельного запуска.
2. **Отслеживание прогресса** — номер последней обработанной строки сохраняется в `/tmp/log_report_last_line`. При следующем запуске обрабатываются только новые строки.
3. **Анализ** — из новых строк извлекаются: top-20 IP, top-20 URL, все HTTP-коды и top-20 ошибок (4xx/5xx с URL).
4. **Временной диапазон** — берётся из timestamps первой и последней обработанной строки.
5. **Отправка** — отчёт отправляется через `mail`. Если утилита недоступна, отчёт выводится в stdout.
