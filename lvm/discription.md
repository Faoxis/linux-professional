### Начало работы с LVM

LVM (Logical Volume Manager) позволяет гибко управлять дисковым пространством, создавать тома, изменять их размеры и выполнять снапшоты. Например, в средах виртуализации, где требуются частые изменения размеров томов или создание резервных копий на лету, LVM предоставляет возможность мгновенно расширять или сжимать дисковое пространство. Это особенно полезно при развертывании новых сервисов, так как можно добавить новый диск и перераспределить пространство без остановки работы системы, а также легко увеличивать или уменьшать тома и восстанавливать данные в случае сбоя.

**Основные компоненты LVM:**

- **Physical Volume (PV):** Физические диски или разделы.
- **Volume Group (VG):** Группы томов, состоящие из одного или нескольких PV.
- **Logical Volume (LV):** Логические тома, создаваемые внутри VG.

**Для работы с LVM необходимы права суперпользователя.**

```bash
sudo -i
```

#### Шаг 1. Проверка доступных устройств

Перед началом работы с LVM необходимо убедиться, что все устройства корректно распознаны системой и доступны для работы. Это позволяет избежать ошибок при создании томов и гарантирует, что нужные диски не используются в других процессах, предотвращая потерю данных на случайно выбранных дисках. Если устройства не будут проверены, это может привести к ошибкам, таким как:

- Неправильное создание томов на дисках с данными, что может привести к потере информации.
- Попытка разметки недоступных или поврежденных устройств, что вызовет сбои при создании LVM структур.
- Использование устройств, занятых другими процессами, что может нарушить работу системы.

Проверка устройств заранее позволяет избежать этих проблем и гарантирует стабильную работу LVM. и доступны для разметки. ### Разметка дисков под LVM

Разметка дисков является важным шагом в настройке LVM, так как она позволяет эффективно использовать доступное дисковое пространство и управлять им. Корректная разметка позволяет избежать ошибок при создании томов и обеспечивает их стабильность и предсказуемость в эксплуатации.

#### 1. Создание Physical Volume (PV)

```bash
pvcreate /dev/sdb
```

**Описание флагов:**

- **pvcreate** – инициализация диска или раздела для использования в LVM.
- **/dev/sdb** – диск, который будет использован как PV.

**Вывод:**

```bash
Physical volume "/dev/sdb" successfully created.
```

#### 2. Создание Volume Group (VG)

```bash
vgcreate otus /dev/sdb
```

**Описание флагов:**

- **vgcreate** – создание группы томов.
- **otus** – имя новой VG.
- **/dev/sdb** – диск, который будет добавлен в VG.

**Вывод:**

```bash
Volume group "otus" successfully created.
```

#### 3. Создание Logical Volume (LV)

```bash
lvcreate -l+80%FREE -n test otus
```

**Описание флагов:**

- **-l+80%FREE** – выделение 80% свободного пространства VG.
- **-n test** – имя нового LV.
- **otus** – VG, из которой выделяется место.

**Вывод:**

```bash
Logical volume "test" created.
```

### Создание файловой системы на LV

```bash
mkfs.ext4 /dev/otus/test
```

**Описание флагов:**

- **mkfs.ext4** – создание файловой системы ext4.
- **/dev/otus/test** – путь к LV.

**Монтирование LV:**

```bash
mkdir /data
mount /dev/otus/test /data
```

**Проверка монтирования:**

```bash
mount | grep /data
```

### Расширение LVM

#### 1. Добавление нового PV в VG

```bash
pvcreate /dev/sdc
vgextend otus /dev/sdc
```

**Описание флагов:**

- **vgextend** – добавление PV в VG.

#### 2. Увеличение LV

```bash
lvextend -l+80%FREE /dev/otus/test
resize2fs /dev/otus/test
```

**Описание:**

- **lvextend** – увеличение размера LV.
- **resize2fs** – изменение размера файловой системы.

---

### Снапшоты

#### 1. Создание снапшота

```bash
lvcreate -L 500M -s -n test-snap /dev/otus/test
```

**Описание флагов:**

- **-s** – создание снапшота.

#### 2. Откат к снапшоту

```bash
lvconvert --merge /dev/otus/test-snap
```

### Уменьшение тома под / до 8G (Пошаговое руководство)

Уменьшение root-раздела - важная задача при перераспределении дискового пространства. Этот процесс включает создание временного тома, перенос данных и восстановление системы на новом томе. Убедитесь, что перед началом работы созданы резервные копии важных данных.

#### Подготовка временного тома для / раздела

Создание временного тома позволит безопасно перенести root-раздел и уменьшить его размер без потери данных. Например, это может потребоваться в случае, если необходимо перераспределить дисковое пространство на сервере с ограниченными ресурсами. Такая операция часто выполняется в виртуализированных средах, где первоначально выделенный объем для root-раздела оказался избыточным и требуется больше места для других разделов, таких как /var или /home. Выполните следующие команды для подготовки временного тома.

```bash
pvcreate /dev/sdb
# pvcreate – инициализация диска для использования в LVM. Эта команда создает физический том (PV), который будет частью Volume Group (VG).
# /dev/sdb – диск, который будет инициализирован как PV. Убедитесь, что диск пуст или содержит данные, которые могут быть удалены.

vgcreate vg_root /dev/sdb
# vgcreate – создание группы томов (VG). VG объединяет один или несколько PV.
# vg_root – имя новой VG.
# /dev/sdb – физический том, который добавляется в VG.

lvcreate -n lv_root -l +100%FREE /dev/vg_root
# lvcreate – создание логического тома (LV) внутри VG.
# -n lv_root – задание имени нового LV (lv_root).
# -l +100%FREE – использование всего доступного свободного пространства VG для нового тома.
# /dev/vg_root – VG, из которой выделяется место для LV.

mkfs.xfs /dev/vg_root/lv_root
# mkfs.xfs – создание файловой системы XFS на логическом томе.
# /dev/vg_root/lv_root – путь к логическому тому, на котором создается файловая система.

mount /dev/vg_root/lv_root /mnt
# mount – монтирование файловой системы в указанную точку.
# /dev/vg_root/lv_root – логический том, который монтируется.
# /mnt – точка монтирования.

xfsdump -J - /dev/VolGroup00/LogVol00 | xfsrestore -J - /mnt
# xfsdump – создание резервной копии файловой системы XFS.
# -J – ведение журнала операций.
# /dev/VolGroup00/LogVol00 – том, с которого создается резервная копия.
# xfsrestore – восстановление файлов из резервной копии.
# /mnt – точка, куда восстанавливаются данные.
```

#### Перенос root на временный том

После подготовки временного тома необходимо скопировать все данные из текущего root-раздела на новый том. Это позволит системе загружаться с временного тома во время уменьшения основного.

```bash
for i in /proc /sys /dev /run /boot; do mount --bind $i /mnt$i; done
# Этот цикл проходит по списку системных директорий /proc, /sys, /dev, /run и /boot.
# mount --bind – связывает указанные директории с их копиями в /mnt. Это необходимо для корректной работы chroot.

chroot /mnt/
# chroot – изменяет корневую файловую систему на /mnt, тем самым позволяя работать в новом root-разделе.
# Все команды, выполняемые после chroot, будут работать в контексте новой системы.

grub2-mkconfig -o /boot/grub2/grub.cfg
# grub2-mkconfig – создает новый конфигурационный файл загрузчика GRUB2.
# -o /boot/grub2/grub.cfg – указывает, куда записать новый конфигурационный файл.
# Это необходимо для обновления параметров загрузки.

cd /boot && dracut -v --force
# dracut – утилита для создания initramfs (начальной файловой системы).
# -v – включает подробный вывод (verbose).
# --force – пересоздает initramfs, даже если он уже существует.
# Этот шаг гарантирует, что новая система сможет загрузиться.
```

#### Уменьшение текущего тома до 8G и возврат root

Теперь основной root-раздел можно безопасно уменьшить. После уменьшения производится перенос данных обратно на основной том и восстановление системы.

```bash
lvremove /dev/VolGroup00/LogVol00
# lvremove – удаляет логический том.
# /dev/VolGroup00/LogVol00 – путь к логическому тому, который будет удален. Удаление освобождает место в группе томов (VG).

lvcreate -L 8G -n LogVol00 /dev/VolGroup00
# lvcreate – создает новый логический том в группе томов.
# -L 8G – указывает размер тома (8 ГБ).
# -n LogVol00 – имя создаваемого логического тома.
# /dev/VolGroup00 – группа томов, в которой создается новый том.

mkfs.xfs /dev/VolGroup00/LogVol00
# mkfs.xfs – форматирует логический том с файловой системой XFS.
# /dev/VolGroup00/LogVol00 – логический том, который будет отформатирован.

mount /dev/VolGroup00/LogVol00 /mnt
# mount – монтирует файловую систему в указанную точку.
# /dev/VolGroup00/LogVol00 – логический том, который монтируется.
# /mnt – точка монтирования, куда будет примонтирован новый том.

xfsdump -J - /dev/vg_root/lv_root | xfsrestore -J - /mnt
# xfsdump – создает резервную копию файловой системы XFS.
# -J – ведение журнала операций.
# /dev/vg_root/lv_root – исходный том, с которого создается резервная копия.
# xfsrestore – восстанавливает данные из резервной копии.
# /mnt – место, куда восстанавливаются данные.
```

### Создание зеркала для /var

Создание зеркала (RAID1) для директории /var обеспечивает отказоустойчивость и защиту данных. RAID1 был выбран, так как он дублирует данные на два диска, что позволяет сохранить информацию даже в случае выхода из строя одного из них. В отличие от RAID0, который увеличивает производительность за счет объединения дисков без отказоустойчивости, RAID1 гарантирует надежность. Альтернативные уровни, такие как RAID5 или RAID10, могут использоваться для баланса между отказоустойчивостью и эффективным использованием дискового пространства, но требуют большего количества дисков. один из дисков выходит из строя, данные остаются доступными благодаря зеркалированию. Следующие шаги описывают создание зеркала и перенос данных.

#### Создание зеркала и перенос данных

```bash
pvcreate /dev/sdc /dev/sdd
# pvcreate – инициализация дисков /dev/sdc и /dev/sdd для использования в LVM.
# Эти диски станут физическими томами (PV), которые позже войдут в группу томов.

vgcreate vg_var /dev/sdc /dev/sdd
# vgcreate – создание группы томов (VG) с именем vg_var, объединяющей /dev/sdc и /dev/sdd.
# Это позволяет создать логический том (LV), используя пространство обоих дисков.

lvcreate -L 950M -m1 -n lv_var vg_var
# lvcreate – создание логического тома (LV) с именем lv_var в группе vg_var.
# -L 950M – размер тома составляет 950 МБ.
# -m1 – создание зеркала (RAID1), где данные будут дублироваться на оба диска.
# -n lv_var – задает имя нового логического тома.

mkfs.ext4 /dev/vg_var/lv_var
# mkfs.ext4 – создание файловой системы ext4 на логическом томе /dev/vg_var/lv_var.
# Это позволит использовать данный том как раздел с файловой системой.

mount /dev/vg_var/lv_var /mnt
# mount – монтирование логического тома в точку /mnt.
# Это позволяет временно использовать том и копировать на него данные.

cp -aR /var/* /mnt/
# cp -aR – рекурсивное копирование данных из каталога /var в /mnt.
# -a – сохраняет права доступа, временные метки и символические ссылки.
# -R – рекурсивное копирование всех подкаталогов и файлов.

mkdir /tmp/oldvar && mv /var/* /tmp/oldvar
# mkdir /tmp/oldvar – создание временного каталога для перемещения старых данных.
# mv /var/* /tmp/oldvar – перемещение всех данных из /var в /tmp/oldvar для освобождения каталога /var.

umount /mnt && mount /dev/vg_var/lv_var /var
# umount /mnt – размонтирование временной точки монтирования /mnt.
# mount /dev/vg_var/lv_var /var – монтирование нового логического тома в /var.

blkid | grep var >> /etc/fstab
# blkid – выводит UUID дисков и разделов.
# grep var – фильтрует строки с информацией о /var.
# >> /etc/fstab – добавляет информацию в /etc/fstab для автоматического монтирования при загрузке системы.
```

### Создание тома для /home

```bash
lvcreate -n LogVol_Home -L 2G /dev/VolGroup00
# lvcreate – создает новый логический том в группе томов VolGroup00.
# -n LogVol_Home – задает имя логического тома (LogVol_Home).
# -L 2G – указывает размер нового тома (2 ГБ).
# /dev/VolGroup00 – группа томов, в которой создается новый том.

mkfs.xfs /dev/VolGroup00/LogVol_Home
# mkfs.xfs – форматирует созданный логический том с файловой системой XFS.
# /dev/VolGroup00/LogVol_Home – логический том, который будет отформатирован.

mount /dev/VolGroup00/LogVol_Home /mnt/
# mount – монтирует логический том в каталог /mnt.
# /dev/VolGroup00/LogVol_Home – логический том, который монтируется.
# /mnt – точка монтирования, куда временно подключается новый том.

cp -aR /home/* /mnt/
# cp – копирует все файлы и каталоги из /home в /mnt.
# -a – сохраняет права доступа, временные метки и символические ссылки.
# -R – рекурсивно копирует все подкаталоги и файлы.
# /home/* – указывает на все файлы и каталоги внутри /home.

rm -rf /home/*
# rm – удаляет файлы и каталоги.
# -r – рекурсивно удаляет все подкаталоги и файлы.
# -f – принудительно удаляет без запроса подтверждения.
# /home/* – удаляет все файлы и каталоги в /home.

umount /mnt
# umount – размонтирует логический том из точки /mnt.

mount /dev/VolGroup00/LogVol_Home /home/
# mount – монтирует логический том LogVol_Home в каталог /home.
# Это позволяет использовать новый логический том как основной каталог /home.

blkid | grep Home >> /etc/fstab
# blkid – выводит UUID всех устройств и файловых систем.
# grep Home – фильтрует строки, связанные с LogVol_Home.
# >> /etc/fstab – добавляет строку с UUID нового тома в /etc/fstab для автоматического монтирования при загрузке системы.
```
