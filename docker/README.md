# Домашнее задание: Docker

## 1) Установка Docker на хост-машину

Инструкция для Ubuntu:
[https://docs.docker.com/engine/install/ubuntu/](https://docs.docker.com/engine/install/ubuntu/)

Проверка установки:

```bash
docker --version
docker run --rm hello-world
```

## 2) Установка Docker Compose

Docker Compose можно использовать как плагин (`docker compose ...`).

Проверка:

```bash
docker compose version
```

## 3) Кастомный образ nginx на базе alpine

В папке задания добавлены:
- `Dockerfile`
- `default.conf`
- `index.html`

`Dockerfile` собран на базе чистого `alpine`: внутри контейнера устанавливается `nginx` через `apk`, подменяется дефолтный конфиг на `default.conf`, затем копируется кастомная страница.

Сборка образа:

```bash
docker build -t faoxis/otus_my_homework:0.01 .
```

Запуск контейнера:

```bash
docker run -d --name custom-nginx -p 8080:80 faoxis/otus_my_homework:0.01
```

Проверка:
- открыть [http://localhost:8080](http://localhost:8080)
- должна отображаться кастомная страница из `index.html`

Остановка и удаление:

```bash
docker stop custom-nginx
docker rm custom-nginx
```

Заливка в docker hub:

```shell
docker push faoxis/otus_my_homework:0.01
```

## 4) Разница между образом и контейнером

- Образ — это неизменяемый шаблон, из которого запускаются контейнеры. 
- Контейнер — это запущенный экземпляр образа с собственным процессом, сетью и томами данных.
- Один образ можно использовать для запуска множества контейнеров.
- Изменения внутри контейнера временные, если не использовать volumes или не собирать новый образ.

## 5) Выводы

Docker позволяет воспроизводимо упаковывать и запускать приложения с одинаковым окружением.

## 6) Можно ли в контейнере собрать ядро?

Да, исходники ядра Linux можно собрать в контейнере как в изолированной среде сборки (при наличии toolchain и ресурсов). 
Кроме того, это популярная практика, которую используют многие Linux Kernel разработчики. 

Важно, что контейнер не имеет собственного ядра: он всегда использует ядро хостовой системы. 
По-сути, сборка ядра в Docker - это более удобная замена chroot.
